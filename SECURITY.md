# Security Policy

### 🚫 Please DO NOT:
- Open a public GitHub issue for security vulnerabilities
- Share the vulnerability publicly before it is resolved

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

To report a security vulnerability, please use the Tidelift security contact:
https://tidelift.com/docs/security

Tidelift will coordinate the fix and disclosure.

## Scope

This policy applies to:
- The main FoodMine application
- API interactions (if applicable)
- Dependencies and configurations

---

## Security Best Practices for Contributors

- Avoid committing secrets (API keys, tokens)
- Keep dependencies updated
- Validate and sanitize user inputs
- Follow Angular security guidelines

---

Thank you for helping keep FoodMine secure! 🙌
